# 학생 출석 관리 시스템

## 프로젝트 개요
외부 서버에서 제공하는 학생 출석 데이터를 받아 화면에 표시한 예제입니다.

## 파일 구성
* server-data-js : 외부 학생 관리 서버에서 제공하는 학생 데이터
* index.html : 메인HTML파일
* style.css : 스타일시트
* script.js : 내부 처리 로직

## 실행 방법
1. 모든 파일을 같은 폴더에 저장합니다.
2. index.html 파일을 웹 브라우저로 엽니다
3. 필터 버튼을 클릭하여 상태별로 학생을 조회합니다.

## 주요 기능
* 외부 학생 관리 서버 데이터를 받아서 학생 카드 목록으로 표시
* 각 학생의ID, 이름, 반, 출석 상태 표시
* 상태별 필터 버튼으로 전체 / 출석 / 결석 / 지각 학생만 조회
* 상태에 따른 색상 표시(전체: 파랑 | 출석: 초록 | 결석: 빨강 | 지각: 노랑)

## 시험 요구사항 설명

### 외부 시스템과 내부시스템
이 예제에서 외부 시스템은 server-data-js 파일입니다. ID, name, classnamw, status 등 학생들의 정보를 저장하고 관리합니다.
내부 시스템은 index.html, style.css, script.js 파일입니다. 외부에서 가져온 데이터를 사용자에게 UI로 변환하여 제공합니다.

### 연계에 사용된 데이터
연계에 사용된 데이터는 외부 시스템 안에 있는 serverStudents 변수에 저장된 학생들의 정보입니다.
배열 형식으로 각 학생의 ID, name, classnamw, status 정보를 포함하고 있으며 내부시스템에서 이 데이터를 연계받아 화화면에 표시합니다.

### 외부 서버 데이터가 웹 화면에 반영되는 흐름
첫번째로 html에서 <script src="server-data.js">를 통해 외부 데이터를 불러옵니다.
두번째로 script.js에 serverStudents 변수를 사용하여 데이터에 접근합니다.
세번째로 displayStudents 변수를 만들어서 학생들의 정보가 담긴 카드를 html에 생성합니다.
네번째로 사용자가 필터 버튼을 누르면 filter() 함수가 조건에 맞는 학생만 골라내도록 합니다.
마지막으로 createElement와 innerHTML을 사용해 데이터 수만큼 카드를 생성하고 appendChild로 추가하여 화면에 표시합니다.
전체적인 흐름을 정리하면 외부 서버 데이터를 내부 시스템에서 받아 필요한 정보를 사용자에게 보여주고 원하는 정보만 필터링 되어 다시 표시하는 것이 됩니다.

### 본 예제가 데이터 연계인 이유
내부 시스템에 직접 입력하지 않고 외부 서버에서 데이터를 가져와 외부와 내부를 연결했기 때문입니다.
외부 서버에 있는 학생 데이터들을 내부 시스템과 연결하여 전송된 데이터를 내부 시스템에서 읽고 화면에 표시합니다. 
이 과정에서 외부 데이터의 내용이 바뀌어도 내부 시스템에서 별도의 수정 없이 자동으로 수정이 됩니다.
복잡한 과정 없이, 사용자의 요청에 따라 실시간으로 데이터를 재구성하여 맞는 데이터로 인터페이스를 구현 및 완성했기 때문에 본 예제가 데이터 연계인 이유가 됩니다.
